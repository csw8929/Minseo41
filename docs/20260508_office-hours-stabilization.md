## Problem Statement

직전 라운드(`feat/offline-position-sync`) 머지 직후, 본인이 매일 쓰면서 거슬리는 작은 항목 5개를 한 라운드에 묶어 정리한다.

1. Settings에 (a) 기본 화질 default, (b) 자막 크기 1x/2x/3x, (c) 회전 잠금 토글이 없다.
2. 재생 중 back을 눌러도 service player가 살아있어 소리가 계속 난다 — 직관에 반함.
3. Firestore 시청 위치 write debounce가 언제 실제로 발사되고 언제 defer되는지 logcat으로 확인할 표준 방법이 정리되어 있지 않다.

## What Makes This Cool

- "내가 다음 영상에서 화질을 또 손볼 일이 없다" — Settings에 default를 묶어 두면 매 영상 진입의 한 번 손이 사라진다.
- 가로 영상에서 폰을 책상에 옆으로 둘 때 의도치 않은 재회전이 사라진다 (회전 잠금 토글).
- back을 누르면 끊기는, 일반적인 안드로이드 영상 앱 멘탈 모델로 회귀.

## Constraints

- 단일 사용자(본인) 개인 앱. ToS 우려로 외부 배포 의도 없음.
- 기존 keystore 호환성 유지 (이전 PC keystore는 의식적으로 무시 — 본 PC 기준).
- 회귀 위험을 최소화 (직전 PR이 hybrid sync 도입이라 player/sync 영역 추가 변경은 신중).

## Premises

1. **기본 화질 default**: 새 영상 진입 시 `selectedMaxHeight`의 시작값을 prefs(`"player"/"defaultMaxHeight"`)로 적용. 영상 내 메뉴 변경은 prefs를 갱신하지 않고 그 영상에서만 유효.
2. **자막 크기**: PlayerView 내장 `SubtitleView.setFractionalTextSize(...)`로 1.0 / 2.0 / 3.0배 적용. prefs key `"player"/"captionScale"`은 **default 값**만 보관. 영상 진입 시 default 적용, 재생 중 in-player 메뉴(자막 메뉴 옆 또는 자막 메뉴 안)에서 변경 가능. **in-player 변경은 prefs에 저장하지 않음** — 그 영상 단위에서만 유효 (화질 메뉴와 동일 패턴). 색·배경은 손대지 않음.
3. **회전 잠금**: prefs key `"player"/"orientationLocked"`. ON일 때 `SCREEN_ORIENTATION_LOCKED`, OFF일 때 `UNSPECIFIED`. 풀스크린 진입 시의 LANDSCAPE 강제 동작은 그대로 유지(잠금보다 풀스크린이 우선).
4. **back→정지/PIP (룰)**: 홈 키·폴드 닫기 = 항상 백그라운드 재생 유지 (composable dispose 안 됨). back 키 = Settings 옵션에 따름:
   - `BACK_ACTION_STOP` (기본): `onDispose` 직전 `mediaController?.pause()`로 정지. controller release. service는 살아있음.
   - `BACK_ACTION_PIP`: `tryEnterPip(activity)` 시도. 성공 시 화면 그대로 PIP 진입(재생 유지). 실패 시 정지로 fallback.
5. **backgroundPlayback 토글 제거**: 본 라운드에서는 토글 동작 wire-up이 아닌 "무조건 정지" 정책. SettingsScreen에서 토글 UI 제거 + prefs key `"backgroundPlayback"` 미사용 처리(완전 삭제는 다음 PR로 미룸 — 사용자 prefs 잔존값 무해).
6. **Firestore 로그**: 이미 존재하는 `Log.d` (`SubFeedPlayerVM`, `SubFeedSync`)에 한 줄 추가 — `onPositionChanged`에서 saveJob을 새로 잡을 때 "defer 30s for videoId=…, positionMs=…". `adb logcat -s SubFeedPlayerVM SubFeedSync`로 흐름 추적.

## Approaches Considered

### Approach A — Minimal (선택됨)

- Settings 3개 토글/리스트 추가 (화질 / 자막 크기 / 회전 잠금)
- back→정지: `onDispose`에서 `pause()` 1줄
- backgroundPlayback 토글 UI 제거
- defer 로그 1줄 + adb 명령 가이드
- 사용자 명시 항목과 1:1 매핑, scope 최소

**Effort: S, Risk: Low**

### Approach B — Settings + Background Wire-Up

- Approach A의 화질/자막/회전 잠금은 동일
- 추가: backgroundPlayback 토글에 의미 부여 (ON이면 back 시에도 재생 유지, OFF면 정지). 홈 키 분기도 같이.
- 사용자가 "단순"을 선택하여 채택하지 않음. 향후 필요해지면 별도 라운드.

## Recommended Approach

**Approach A** — 사용자 선택. 동작 예측 가능성과 코드 단순성 우선. backgroundPlayback 토글은 의미가 비어 있는 상태로 남기느니 UI에서 제거.

### 구현 단계

1. **SettingsScreen / SettingsViewModel 정비**
   - `prefs("player")` 에 `defaultMaxHeight: Int`, `captionScale: Float`, `orientationLocked: Boolean` 추가
   - 기존 `backgroundPlayback` 토글 UI 제거 (prefs key는 일단 그대로 두고 다음 PR에서 정리)
   - 화질 리스트는 `availableQualityHeights`가 아닌 고정 후보 (0=자동, 1080, 720, 480, 360). 영상 트랙에 없으면 ExoPlayer가 자동 fallback.

2. **PlayerViewModel 진입점 수정**
   - `loadVideo` 시작부에서 `selectedMaxHeight = prefs.getInt("defaultMaxHeight", 0)` 적용
   - 영상 내 메뉴에서 `selectQuality` 호출 시 prefs 갱신 안 함 (영상 단위 유지)

3. **자막 크기 — default + in-player override**
   - **PlayerViewModel**에 `selectedCaptionScale: Float` UiState 필드 추가. `loadVideo` 시작부에서 `prefs.getFloat("captionScale", 1.0f)` 로 초기화.
   - **PlayerViewModel.selectCaptionScale(scale: Float)** 함수 추가 — UiState만 갱신, prefs 저장 없음 (영상별 override).
   - **PlayerScreen** `AndroidView { PlayerView(...) }`의 `update` 블록에서 `view.subtitleView?.setFractionalTextSize(0.0533f * uiState.selectedCaptionScale)`. 0.0533은 Media3 기본 비율. `update`는 state 변경마다 호출되므로 in-player 변경이 즉시 반영됨.
   - **In-player UI**: 기존 `CaptionMenu` 옆에 자막 크기 토글(1x/2x/3x cycle 버튼) 또는 `CaptionMenu` 하단에 항목 추가. 본인 선호로 결정.
   - **SettingsScreen**: 1x/2x/3x 라디오 또는 SegmentedButton으로 default 저장.

4. **회전 잠금**
   - `PlayerScreen` 의 `LaunchedEffect(uiState.isFullscreen)` 옆에 `LaunchedEffect(orientationLocked)` 추가하여 활성 시 `SCREEN_ORIENTATION_LOCKED`, 비활성 시 `UNSPECIFIED` (단, 풀스크린 enter 시는 풀스크린 LANDSCAPE 우선).
   - 풀스크린 exit 시에는 `orientationLocked` 값에 따라 LOCKED / UNSPECIFIED 복귀.

5. **back→정지**
   - `PlayerScreen.kt:74` `DisposableEffect(context) { ... onDispose { mediaController?.pause(); mediaController?.release(); ... } }` 로 변경. `pause()`는 release 전에 호출되어야 controller가 살아있는 상태에서 service player에 명령이 전달됨.
   - service의 `onTaskRemoved` 정리 정책은 건드리지 않음.

6. **로그 보강 + 가이드**
   - `PlayerViewModel.onPositionChanged` 시작부에 `Log.d(TAG, "defer scheduled (30s) — videoId=$currentVideoId, positionMs=$positionMs")` 추가.
   - `saveJob.cancel()` 직후 (즉 새 schedule 잡힐 때마다 이전 scheduled 가 cancel됨)을 의식한 로그.
   - `docs/development-log.md` 또는 README에 `adb logcat -s SubFeedPlayerVM SubFeedSync` 명령 + 예상 출력 sample 추가.

## Open Questions

- **자막 크기 단위**: 사용자 명시 "1x/2x/3x". 2x/3x가 실제로 가독성 OK인지, 너무 큰지는 단말 검증 필요. 너무 크면 1.0 / 1.5 / 2.0 으로 후속 조정.
- **자막 크기 in-player UI 위치**: 기존 `CaptionMenu` 안에 sub-section으로 둘지, `PlayerBottomBar`에 별도 토글 버튼으로 둘지. 본 디자인은 `CaptionMenu` 통합을 가정 (메뉴 진입 횟수가 화질보다 적기 때문).
- **화질 후보 리스트**: 0/1080/720/480/360 외에 4K(2160) 옵션 필요? 본인 단말이 4K 재생 가능한지 확인 필요.
- **회전 잠금 적용 범위**: PlayerScreen만? 아니면 앱 전역 (FeedScreen 포함)? 본 디자인은 PlayerScreen만 가정.

## Success Criteria

- Settings에서 3개 항목 모두 토글/선택 가능, 앱 재시작 후에도 유지.
- back 후 1초 내 소리 정지 확인 (logcat `SubFeedPlayerVM` 에 `savePositionNow` 로그 + service player.isPlaying=false).
- `adb logcat -s SubFeedPlayerVM SubFeedSync` 만으로 30초 debounce flow가 시간 순으로 읽힌다 — defer schedule → cancel(=새 defer) → 발사 → Firestore ok.
- 자막 크기 3x에서 폴드 메인 + 폰 둘 다 화면 잘림 없이 표시.

## What I noticed about how you think

- **UI에 일관성을 우선시함**: D3에서 "토글에 동작을 붙이는 게 깔끔"이라는 추천을 받았음에도 "단순"을 선택. 이유는 분명히 말씀하시지 않았지만, 토글이 UI에만 남고 의미가 비어 있는 상태가 더 거슬린다는 판단으로 읽힙니다 — 동작 vs UI 둘 중 하나가 깔끔하면 충분이라는 결.
- **본인이 사용자**라는 점이 직접 발화에 반영됨. "추가하고 싶은 것"의 1.1/1.2/1.3 모두 본인이 매일 마주하는 마이크로 마찰 (기본 화질, 자막 크기, 회전). 가설이 아니라 관찰.
- **로그 가시성을 묻는 본능**이 있음. 직전 PR에서 Firestore write debounce 가 들어갔는데, "잘 동작하나" 가 아니라 "확인하는 표준 방법"을 물으셨습니다 — 운영자 마인드.
- **scope를 작게 끊는 감각**: "이것저것 + 작은 기능"이라고 시작했지만 막상 항목은 5개 + 모두 같은 영역(Settings / Player)에 모여 있어 한 PR로 묶기 적합. 자연스럽게 응집된 라운드를 골라내심.
