## Problem Statement

현재 `ui/PlayerScreen.kt`는 Media3 `PlayerView` 기본 컨트롤(`useController` 기본값=true)에 의존 중. 사용자는 일반 비디오 플레이어 수준의 UX를 원함.

확인된 부족점:
- 전체화면 전환 없음 (앱 portrait 고정, 가로 회전 안 됨)
- 화질 선택 없음 (HLS adaptive면 ExoPlayer 자동 결정에만 맡김)
- ±30초 스킵 버튼 없음
- 컨트롤 버튼이 작음 (Material 기본 사이즈)
- "다른 단말 이어보기" — `PlayerViewModel` + `SyncRepo`에 코드는 있으나 Google Sign-In 플로우가 없어서 `auth.currentUser`가 항상 null → `SyncRepo.savePosition()` 첫 줄 `val uid = uid ?: return`에서 silent skip → Firestore에 아무것도 저장 안 됨 → 사실상 미동작
- PIP / 백그라운드 재생 없음
- 자막 없음

## What Makes This Cool

- Minseo21 (libVLC 기반 "삿치" 플레이어)에서 검증된 컨트롤 패턴 — 가운데 가로 3등분 큰 버튼, 토글 가능한 상단/하단 그라디언트 — 을 Compose로 이식
- 양면 더블탭 ±30초 (YouTube/Netflix 패턴) — 버튼 안 눌러도 빠르게 점프
- 한 라운드에 일반 비디오 앱이 갖춘 기능을 정리: 전체화면 + 화질 + 자막 + PIP + 백그라운드 + cross-device resume
- 기존 도메인 가치(광고 없는 YouTube 구독 피드)는 그대로 두면서 재생 경험만 일반 플레이어 수준으로 끌어올림

## Constraints

- 개인 사용 / 배포 없음
- Android-only, min SDK 26 / target 36
- **Compose-only** — XML 도입 금지 (프로젝트 일관성)
- ExoPlayer (Media3 1.5.1) 유지
- 현재 동작 중인 iOS InnerTube 스트림 추출 흐름(`NewPipeVideoExtractor.getStreamUrl`) 유지
- 화질 선택은 HLS adaptive variants에 한정 (muxed URL fallback 시 비활성)
- Firebase 의존성 그대로 유지 (`firebase-auth`, `firebase-firestore`)

## Premises

1. **화질 선택은 HLS variants에서만 활성** — InnerTube가 HLS manifest 반환할 때만 메뉴에 여러 해상도 표시. Muxed `formats[]` fallback일 땐 단일 해상도 고정 — 메뉴 회색 처리.
2. **자막은 InnerTube 응답에서 추출** — `captions.playerCaptionsTracklistRenderer.captionTracks[]`에 트랙 목록과 timedtext URL이 있음. XML → SRT 간단 변환 후 ExoPlayer `MediaItem.SubtitleConfiguration`로 add. 외부 자막 API/라이브러리 불필요.
3. **PIP/백그라운드는 manifest 변경 + Media3 MediaSessionService** — `media3-session` 의존성은 이미 있음. `MainActivity` manifest에 `supportsPictureInPicture`, `resizeableActivity`, `configChanges` 추가 + 새 `SubFeedMediaSessionService` 등록.
4. **`PlayerView` 기본 컨트롤은 끄고 Compose 오버레이로 직접 그림** — `useController=false`. 모든 컨트롤은 Compose Box.
5. **Google Sign-In 없으면 cross-device resume이 silent하게 안 됨** — `auth.currentUser==null`일 때 `SyncRepo`가 noop. 활성화 조건은 사용자가 한 번 Google 계정으로 로그인하는 것. 익명 인증은 단말마다 uid가 달라져 cross-device가 안 되므로 부적합.

## Approaches Considered

### Approach A: AndroidView(PlayerView) + Compose 오버레이 (선택)
- `AndroidView`로 `PlayerView(it).apply { useController = false; player = exoPlayer }`만 띄우고, 모든 컨트롤은 Compose `Box(Modifier.fillMaxSize())` 오버레이
- 양면 더블탭 = `Modifier.pointerInput { detectTapGestures(onDoubleTap = ...) }` 좌/우 영역 두 개
- 컨트롤 토글은 `var visible by remember { mutableStateOf(true) }` + `LaunchedEffect`로 5초 후 숨김
- Effort: 인간 ~1주 / CC ~45-60분
- Pros: Compose-first 일관성, 제스처/애니메이션 표현 자연스러움, ExoPlayer 직접 조작 자유
- Cons: AndroidView ↔ Compose 경계 한 군데 — 단순한 경계라 큰 부담 아님

### Approach B: Custom XML controller (Media3 표준 controller customization)
- `PlayerView`에 `app:controller_layout_id="@layout/exo_custom"` 지정, Minseo21의 `activity_main.xml`을 거의 그대로 이식
- Media3가 SeekBar 자동 동기화/시간 표시 등 일부 기능 무료로 제공
- Effort: 인간 ~1주 / CC ~40분
- Pros: Minseo21 자산 재활용 폭이 큼, Media3 내장 기능 공짜
- Cons: SubFeed는 Compose-only 프로젝트인데 XML 섞임 (FeedScreen/SettingsScreen은 Compose). 양면 더블탭 같은 커스텀 제스처는 PlayerView 위 별도 OnTouchListener — Compose 대비 표현력 떨어짐.

## Recommended Approach

**Approach A — Compose 오버레이.**

Approach B의 "Media3 내장 컨트롤 공짜" 이점은 우리가 컨트롤을 모두 새로 그리기로 결정한 시점에 무의미해짐. 일관성 비용이 이득보다 큼.

### 컨트롤 레이아웃 (Compose 오버레이)

```
┌──────────────────────────────────────────────┐
│ ← title              [HD][자막][⛶][⋮]      │ ← TopBar (그라디언트, 토글)
├──────────────────────────────────────────────┤
│                                              │
│   ⏪ -30s     ▶ Play/Pause     ⏩ +30s        │ ← 가로 3등분, 각 33%
│   (큰 버튼)    (큰 버튼)       (큰 버튼)      │   터치 영역
│                                              │
├──────────────────────────────────────────────┤
│ 0:32                            8:24    [⛶] │
│ ──────●─────────────────────────────────     │ ← Slider
└──────────────────────────────────────────────┘
                                                  ↑ BottomBar (그라디언트, 토글)
```

- 단일 탭 → 전체 컨트롤 visibility 토글 (보일 땐 5초 후 자동 숨김, 일시정지 시엔 계속 표시)
- 컨트롤 visible 시에만 인터랙션 가능 — invisible 시 영역 자체 비활성

### 양면 더블탭 ±30초

```
┌──────────────┬──────────────┐
│   좌 핫존     │   우 핫존     │
│  더블탭 시    │  더블탭 시    │
│   -30초      │   +30초      │
│  ripple FX   │  ripple FX   │
└──────────────┴──────────────┘
```

- 두 개의 `Box(Modifier.fillMaxHeight().weight(1f).pointerInput { detectTapGestures(onDoubleTap = ...) })`
- 단일 탭은 컨트롤 토글로 위임 (`detectTapGestures`의 `onTap` 분리)
- 더블탭 ripple — `Animatable<Float>` + `drawWithCache`로 원형 ripple

### 상단 바
- 좌: `IconButton` 뒤로가기 + `Text` (영상 title, ellipsize, 한 줄)
- 우: 화질 pill (`HD` / `1080p` / `Auto`) → 탭 시 `QualityMenu` BottomSheet
- 우: 자막 토글 아이콘 → 탭 시 `CaptionMenu` BottomSheet (사용 가능 트랙 목록 + Off)
- 우: PIP 진입 아이콘 (`Icons.Default.PictureInPicture`)
- 우: 옵션 메뉴 (overflow) — 백그라운드 재생 ON/OFF 등

### 하단 바
- 좌: 현재 시간 (`mm:ss`)
- 중앙: `Slider` (Material3) — 드래그 시 `exoPlayer.seekTo`, 드래그 중엔 thumb 위에 미리보기 시간 라벨
- 우: 총 시간
- 우 끝: 전체화면 토글 (`Icons.Default.Fullscreen` / `FullscreenExit`)

### 화질 선택
- HLS manifest 로드 후 `exoPlayer.currentTracks.groups`에서 video `TrackGroup` 찾음 (`type == TRACK_TYPE_VIDEO`)
- 각 `Format`의 `width × height`를 라벨화 (1080p / 720p / 480p / 360p)
- BottomSheet: `[자동, 1080p, 720p, 480p, 360p]`
- 선택 시 `exoPlayer.trackSelectionParameters = exoPlayer.trackSelectionParameters.buildUpon().setMaxVideoSize(w, h).build()`
- HLS가 아니면 (URL이 `hls:` prefix 없음) pill 회색 + tap 시 short snackbar "단일 해상도만 지원"

### 자막
- `NewPipeVideoExtractor.getStreamUrl()` 변경 — InnerTube 응답에서 `captions.playerCaptionsTracklistRenderer.captionTracks` 추출 후 `Triple<streamUrl, captionTracks, ...>` 같은 형태로 반환 (인터페이스 시그니처 변경 또는 별도 메서드 `getCaptions(videoId)` 분리)
- `CaptionTrack(languageCode, displayName, baseUrl, isAutoGenerated)` 데이터 클래스
- `TimedTextToSrt.convert(xmlString): String` — XML `<text start="..." dur="...">` → SRT `00:00:00,000 --> 00:00:00,000\ntext\n\n`
- `MediaItem.Builder().setSubtitleConfigurations(listOf(SubtitleConfiguration.Builder(srtUri).setMimeType(MimeTypes.APPLICATION_SUBRIP).setLanguage("ko").build()))`
- 자막 토글 ON 시 `exoPlayer.trackSelectionParameters` 의 `setPreferredTextLanguage("ko")`
- 우선순위: 한국어 자동(asr) > 한국어 원본 > 영어 자동 > 사용자 선택

### 전체화면 전환
- 토글 시:
  - `(context as Activity).requestedOrientation = ActivityInfo.SCREEN_ORIENTATION_LANDSCAPE`
  - `WindowInsetsControllerCompat(window, decorView).hide(WindowInsetsCompat.Type.systemBars())`
- 다시 토글:
  - `requestedOrientation = SCREEN_ORIENTATION_PORTRAIT`
  - insets show
- 시스템 back 버튼은 전체화면일 때 portrait 복귀 우선, 아닐 때 `navController.popBackStack()`

### PIP
- `MainActivity` `AndroidManifest.xml`:
  ```xml
  <activity
      android:name=".MainActivity"
      android:supportsPictureInPicture="true"
      android:resizeableActivity="true"
      android:configChanges="screenSize|smallestScreenSize|screenLayout|orientation" />
  ```
- PIP 진입: `PictureInPictureParams.Builder().setAspectRatio(Rational(16, 9)).build()` → `enterPictureInPictureMode(params)`
- `MainActivity.onPictureInPictureModeChanged(isInPip, _)` → ViewModel state로 전달 → 컨트롤 visible=false 강제

### 백그라운드 재생
- `SubFeedMediaSessionService : MediaSessionService`:
  ```kotlin
  override fun onCreate() {
      super.onCreate()
      val player = ExoPlayer.Builder(this).build()
      mediaSession = MediaSession.Builder(this, player).build()
  }
  ```
- `AndroidManifest.xml`에 `<service>` + `MEDIA_SESSION` intent-filter 등록
- `PlayerScreen`은 `MediaController` 통해 player 받음 (직접 ExoPlayer 인스턴스화 X) — 화면 떠도 service의 player 살아있음
- 알림 컨트롤은 `MediaSessionService`가 자동 노출

### Google Sign-In + Resume banner
- `AuthRepo`:
  ```kotlin
  class AuthRepo @Inject constructor(
      private val auth: FirebaseAuth,
      @ApplicationContext context: Context,
  ) {
      val currentUser: Flow<FirebaseUser?> = callbackFlow { ... }
      fun signInIntent(): Intent  // GoogleSignInClient.signInIntent
      suspend fun handleSignInResult(data: Intent?): Result<FirebaseUser>
      suspend fun signOut()
  }
  ```
- `SettingsScreen`에 새 섹션 "Google 계정":
  - 미로그인: `Button("Google 계정 연동")` → `MainActivity` `ActivityResultLauncher` launch
  - 로그인됨: `"{email} ✓"` + `Button("로그아웃")`
- `MainActivity`에 `rememberLauncherForActivityResult`로 sign-in result 처리 → `SettingsViewModel.handleSignInResult(data)`
- `PlayerViewModel`:
  - `loadVideo()`에서 `resumePositionMs > 0L`이면 `_uiState.update { it.copy(showResumeBanner = true) }`
  - `dismissResumeBanner()` / `restartFromBeginning()` (= `seekTo(0)` + dismiss)
- `ResumeBanner` Compose:
  - 영상 시작 5초간 화면 상단(또는 영상 위) 작은 카드: `"X:XX부터 이어봅니다"` + `[처음부터]` 버튼 + `[X]` dismiss
  - 5초 후 자동 dismiss

## Open Questions

1. **PIP 진입 트리거** — 사용자가 명시적으로 PIP 버튼 누를 때만 진입할지, 홈버튼 누르면 자동 진입할지. 자동은 user surprise 가능. → **명시 진입을 default**로, 자동은 옵션 메뉴에 토글로 별도 라운드.
2. **자막 fallback 우선순위** — ko 자동(asr) → ko 원본 → en 자동 → 사용자 선택. ko 원본이 ko 자동보다 우선이어야 할까? 자동(asr)은 음성 인식 품질이 떨어지므로 원본 우선이 합리적.
3. **회전 잠금 옵션** (Minseo21에 있던 항목) — 이번 design에 미포함. 전체화면 토글로 충분한지, 자유 회전 잠금이 별도로 필요한지는 사용 후 판단.
4. **MediaSessionService 도입이 cross-screen navigation을 깨뜨릴 가능성** — MediaController 비동기 연결 중에 화면 전환 시 race condition. 첫 번째 구현 후 직접 테스트 필요.

## Success Criteria

- [ ] PlayerScreen이 영상 위에 가운데 3등분 큰 컨트롤(±30초 / Play / +30초)을 토글 가능하게 표시
- [ ] 화면 좌/우 더블탭으로 -30초/+30초 점프 + ripple 시각 피드백
- [ ] 화질 선택 메뉴가 HLS manifest일 때 활성, muxed일 때 비활성 (시각적 구분)
- [ ] 자막 ON 시 영상 위에 timed text 표시 (한국어 원본 우선, 없으면 ko 자동, 없으면 사용자 선택)
- [ ] 전체화면 토글 시 가로 회전 + 시스템 바 hide, 다시 토글 시 portrait 복귀
- [ ] PIP 버튼 → 작은 윈도우로 이어 재생, 알림 컨트롤로 일시정지 가능
- [ ] 옵션 메뉴 "백그라운드 재생" ON 시 홈/잠금 화면에서 음성만 이어 재생
- [ ] Settings에서 Google 로그인 후, 폰에서 5분 시청 → 태블릿에서 같은 영상 열면 Resume banner + 5분부터 재생
- [ ] [처음부터] 탭 시 `seekTo(0)` + banner dismiss
- [ ] 모든 컨트롤 버튼 최소 56dp × 56dp (Material 권장 48dp 초과 — "큰 버튼" 요구)

## What I noticed about how you think

- "기본 6개 + 더 추가할 만한 거 제안"이라고 열어둔 자세 — 본인 안의 요구를 다 꺼내는 데 거리낌이 없음. 보통은 "이 정도면 됐다"에서 멈춤.
- 추가 제안 4개 중 재생 속도(A)는 빼고 PIP/자막/더블탭은 골랐음. 사용 패턴이 자연스럽게 드러남 — 음악/팟캐스트가 아니라 영상 시청자, 길게 본다, 멀티태스킹 한다.
- "다른 단말 이어보기"를 design 거의 끝에 다시 꺼낸 것 — design.md에 명시되어 있는 항목인데 실제로 동작 안 하는 상태였고, 그걸 잡아낸 게 정확히 design doc과 코드 사이에서 가장 잘 깨지는 지점. "있다고 적혀 있으니 있을 것"으로 넘기지 않고 다시 점검하는 습관이 있음.
- Minseo21을 참고 자료로 명시한 것 — 이미 작동 중인 자기 코드를 재활용하려는 실용주의. 다른 사람의 코드를 베끼는 게 아니라 본인이 만들어둔 패턴을 옮기는 거라 재학습 비용이 거의 0.
